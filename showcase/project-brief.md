# Project Brief

## What is this project?

This is a document-driven AI language learning system that runs on top of general-purpose AI tools.

Instead of building a standalone app, the project designs a structured workflow (a "harness") that guides AI behavior into a more stable learning system.

## What problem does it solve?

Current AI usage in language learning is often:

- fragmented;
- inconsistent;
- hard to track over time;
- overly dependent on translation instead of understanding.

Many learners are interested in foreign-language books, games, music, films, or online content, but they often get stuck at the level of "translate when confused" rather than building durable comprehension.

## What is the solution?

The system introduces:

- clear session boundaries;
- structured onboarding;
- controlled AI behavior rules;
- learning records and report anchors;
- continuation paths across conversations;
- user-facing simplification over an internally structured document system.

## Why not build an app first?

The current stage focuses on validating whether a structured AI workflow can produce stable learning behavior using existing general-purpose AI tools.

This avoids prematurely building a full application before the learning loop, product logic, and user-facing path are clear enough.

## Current stage

The project is in validation and runtime packaging. The priority is to make the system understandable, testable, and reviewable before deeper commercialization or technical implementation.

## What should reviewers look at first?

Start with:

1. `README.md` at the repository root.
2. `showcase/demo-path.md`.
3. `showcase/reviewer-guide.md`.

The deeper `docs/` and `runtime/` folders can be added as the internal system documents are prepared for public inspection.
