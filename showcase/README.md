# Showcase

This folder is for public-facing materials that help reviewers, judges, collaborators, or portfolio visitors quickly understand the project.

## Recommended reading order

1. `project-brief.md` — a short explanation of the project, problem, solution, and current status.
2. `demo-path.md` — a simple path for understanding how the system may be experienced.
3. `reviewer-guide.md` — guidance for reviewers who want to inspect the repository without getting lost in internal design documents.

## Role of this folder

The system itself may contain complex internal rules and runtime documents. This folder should remain simple, readable, and external-facing.
